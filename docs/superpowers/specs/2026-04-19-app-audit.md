## 🔴 Critical

1. **SQL injection** — `activity.js:37–42` string-concats `state.topic` / `state.kind` into WHERE clauses. Fix with parameterized `:topic` / `:kind`.
2. **BYOK cache TTL mismatch** — `api.js:158` caches `byokStatus` 30 s but `byokSet()` doesn't invalidate it; stale key state leaks into next collect.
3. **Dashboard fails hard on one 500** — `home.js:427` `Promise.all([listTopics, overviewStats])` rejects on either failure; no `.catch`.
4. **solutions.js:190** swallows extraction errors silently.

## 🟠 Performance

1. **`listTopics` re-fetches per nav** — `main.js:113–122` fires it on every DOMContentLoaded to populate nav count. No module-level singleton. Dashboard → Activity → Settings = 3 spawns of the same query.
2. **`detectExistingSources` not memoized** — `topic.js:120` re-runs every time the Source Picker modal opens for the same topic.
3. **Freshness poller runs when hidden** — `api.js:110–140` polls `db_mtime` every 5 s whether the app is frontmost or backgrounded. 720 needless calls/hour when tab is hidden.
4. **`AGGRESSIVE_SOURCES` not validated** — `collect.js:200` hardcoded list; if a source name changes in Python, chips hang.

## 🟡 UX rough spots

1. **Collect error has no retry** — `collect.js:80–83` "Open gap map" button is hidden on error; no alternative path.
2. **Topic errorCard buttons unwired** — `topic.js:75–89` renders buttons but never attaches `onclick`.
3. **Activity pagination spam** — `activity.js` event listeners re-wired per render; rapid-click fires concurrent queries.
4. **Search has no loading state** — `search.js:63–78` user sees frozen UI during 10 s PRAW call.
5. **Find "not ready" link not clickable** — `find.js:78–85` text says "go to Settings" without a link.
6. **Database console has no syntax help** — `database.js:98` raw textarea; cryptic SQL errors only after full run.

## Top 5 cross-cutting fixes

1. **Module-level `listTopics` cache** (main.js) — one fetch per app session, invalidate on `collect:done`.
2. **`byokSet` → invalidate `byok_status` cache** (api.js) — fresh key state next call.
3. **Parameterize activity.js queries** — match database.js pattern.
4. **Promise.all → Promise.allSettled on dashboard** — partial success beats total failure.
5. **Visibility gate on freshness poller** — `if (document.visibilityState !== 'visible') return`.
