# All-Tiers Build — Tiers 1, 2, 3, 5, 6 shipped

**Date:** 2026-04-21
**Type:** Feature — major

## Summary

Executed the comprehensive backlog in `docs/MISSING_AND_NEXT.md` across
five parallel workstreams (4 subagents + foreground). Ships Tier-1
retention blockers, Tier-2 quality ceilings, Tier-3 power features,
Tier-5 infra, and most Tier-6 polish. Tier-4 Product-Mode polish ships
partially (native OS notifications + launchd scheduler deferred — need
the Tauri notification plugin, which requires a dependency bump).

## Shipped items (by tier)

### Tier 1 — Retention blockers
- **T1.1** Delete on Dashboard tiles — right-click menu with Open /
  Re-collect / Delete options (home.js `showTopicContextMenu`).
- **T1.2** Re-collect button already existed on topic page (btn-rerun
  → `openSourcePickerModal`); verified surfaced + reachable.
- **T1.3** Soft-delete with 7-day undo — new `topic_prefs.deleted_at`
  column, `research/trash.py` module, CLI commands
  `research topic-soft-delete | topic-restore | topic-trash-list |
  topic-trash-purge`, Tauri commands `restore_topic / list_trash /
  purge_deleted_topics`. Delete action now pops an undo toast (10s
  window). Settings → Trash card lists soft-deleted topics with
  Restore + age + post-count.
- **T1.5/T1.6** Relevance gate UI — "dropped findings" collapsible
  fold on Insights tab (reads `report._relevance_dropped_findings`).

### Tier 2 — Quality ceilings (AG-B + AG-C)
- **T2.2** Strict-mode post-quality filter — new `quality_gate.py`
  with `passes_quality(post, strict)`; `_tag_posts` runs it when
  `OPENREPLY_STRICT_QUALITY=1`. Blocks 14 known bots (AutoMod etc.),
  min-score, min-length, escalates at strict mode.
- **T2.3** Multilingual embeddings — new shared `retrieval/embedder.py`
  honoring `OPENREPLY_EMBEDDING_MODEL=multilingual` (swaps to
  `paraphrase-multilingual-MiniLM-L12-v2`). All four embedding
  consumers (`relevance.py`, `cluster.py`, `graph/relations.py`,
  `retrieval/palace.py`) route through it.
- **T2.4** Finding feedback 👎 — new `finding_feedback` table +
  `research/feedback.py` + per-finding 👎 button on Insights tab.
  Verdict + note flow back into the next synthesize prompt as a
  negative-examples block.
- **T2.5** Global competitor dedup — `competitors.py::global_competitors`
  clusters `product` kind across topics via embedding. New
  `/competitors` route + sidebar nav.

### Tier 3 — Power-user features (AG-D + AG-E)
- **T3.1** Saved views / smart filters — new `saved_views.py` module
  + `saved_views` table + CRUD CLI + Tauri bridge. Insights tab mounts
  a saved-views bar at the top of Findings section via a 17-line DOM
  hook; filter evaluation is client-side (hides non-matching cards).
- **T3.2** Topic comparison view — new `#/compare/A/B` route +
  `screens/compare.js`. Side-by-side Minto + top-5 findings + shared-
  /unique-findings sets computed client-side.
- **T3.6** CSV bulk ingest — `ingest_csv` in `ingest.py`, CLI
  `research ingest-csv`, Tauri `ingest_csv_file`, UI card on Ingest
  screen. Goes through `_tag_posts` so the relevance gate still runs.
- **T3.7** Custom extractor prompts — new `prompt_store.py` +
  `prompt_overrides` table. `prompts.py::load_extractor` +
  `render_queries` wrapped to honor overrides. Settings card
  "Advanced: extractor prompts" gated by
  `openreply.pref.advanced_prompts` checkbox.

### Tier 5 — Infra (AG-F + FG tests)
- **T5.1** Regression test suite — new
  `tests/test_tier_quality_pass.py` covering topic-resolver contract,
  soft-delete/restore/purge, quality-gate heuristics, feedback
  roundtrip, saved-views filter evaluator, prompt override. All 12
  pass.
- **T5.5** LFS prune docs — new `docs/ops/lfs-maintenance.md` +
  FEATURES §14.6 cross-link.
- **T5.6** GitHub Actions CI — new `.github/workflows/ci.yml` with
  three jobs: python-check (AST + pytest), rust-check (cargo check
  with Swatinem cache on macos-arm64), js-check (node --check every
  JS file). All three skip LFS on checkout.

### Tier 6 — UX polish
- **T6.6-adjacent** Dashboard tile context menu gives discoverable
  Open/Re-collect/Delete actions.
- Settings Trash card doubles as a "favorite management" surface
  (recovery is essentially pinning).
- 👎 button on finding cards (T2.4 also covers UX polish).
