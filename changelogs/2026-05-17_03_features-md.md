# Add root-level FEATURES.md feature catalog

**Date:** 2026-05-17
**Type:** Documentation

## Summary

Created the mandatory root-level `FEATURES.md` — the single source of truth
for every user-facing feature, its flow, code location, completeness state,
and known gaps. Built from a CodeGraph sweep of all 16 feature categories
across the MCP server, CLI, and Tauri desktop app.

## Changes

- Catalogued 182 features across 16 categories: data fetching (33 source
  adapters), discovery/collection, corpus management, synthesis/gap finding,
  knowledge graph, semantic search, persona agents, paper pipeline, product
  tracking, audience/competitors, export, MCP server/jobs, CLI, advanced
  analysis modules, the Tauri desktop app, and customization/feedback.
- Every feature carries `file:line` citations verified against the codebase
  (147 MCP tools mapped via AST extraction of `mcp/server.py` and
  `mcp/tools/persona_tools.py`; implementation modules mapped from
  `research/`, `sources/`, and `persona/`).
- Status reconciles: 157 ✅ complete, 25 🟡 partial, 0 🚧, 0 ❌.
- Known-gaps rollup surfaces the P0 launch blockers (stale Apr-21 sidecar
  binary, missing Developer ID cert, missing `JWT_DESKTOP_SECRET`) and the
  P1/P2 items (14 advanced modules with no MCP tool, 11 unfinished Tauri
  screens, no persona test coverage).
