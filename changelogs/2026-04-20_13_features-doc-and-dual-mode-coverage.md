# Features & usage guide + honest Dual-Mode coverage map

**Date:** 2026-04-20
**Type:** Documentation

## Summary

Creates `docs/FEATURES.md` — a single comprehensive reference for everything
shipped across ROADMAP Phases 1–11, with detailed usage instructions for every
surface (CLI + Tauri IPC + UI navigation), keyboard shortcuts, provider notes,
and schema summaries.

The doc ends with an honest §15 "Coverage vs Dual-Mode Pivot" section that
answers: *does shipping all 11 phases cover DUAL_MODE_PIVOT.md?* The answer is
**no** — what we built is the Topic Mode enhancement roadmap. Product Mode
(DUAL_MODE_PIVOT §4) is a separate ~5-month build (Phases A–G) that requires
Product/Competitor/ConnectedSource/Signal entities, OAuth integrations, account
system + billing, scheduled per-product sweeps with typed signals, and a
Daily Dashboard with five dedicated sections (Mirror / Lens / Field / Signals /
Hypotheses).

§15.1 lists the 7 shipped phases that provide groundwork Product Mode can lean
on. §15.2 lists the 11 DUAL_MODE_PIVOT requirements that are NOT shipped. §15.4
recommends running the 2-week validation experiment with 3 founders before any
Product Mode build kicks off.
