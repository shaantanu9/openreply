# Release v0.1.21 — chat decomposition, RSS feeds, collect fix

**Date:** 2026-06-05
**Type:** Infrastructure | Feature | Fix

## Summary

Cut desktop release **v0.1.21** (the previous tag `v0.1.20` predates 17 commits
of merged work). Bumps the app version across `package.json`, `tauri.conf.json`,
and `Cargo.toml`/`Cargo.lock`, then ships via the tag-driven release pipeline
(`release-mac` / `release-windows` / `release-linux`), which signs + notarizes
and promotes draft → published. The website's `/download` resolves GitHub
`releases/latest`, so it serves v0.1.21 automatically once promoted.

## What ships in v0.1.21 (vs v0.1.20)

- **Chat backend decomposition** — `chat.py` → tested 7-module package
  (`source_intent`, `timeout`, `llm_dispatch`, `retrieval_context`,
  `agent_tools`, `meta`, `doctor`); 32 unit tests.
- **Reusable chat UI** — `screens/chat/chatPanel.js` (`mountChatPanel`) +
  shared `chatState.js` + `lib/markdown.js`; topic Chat tab and the sidebar
  Chats list now mount the SAME component.
- **Sidebar inline chat** — clicking a saved chat opens inline (no jump to the
  topic Chat tab).
- **`chat doctor`** — one-call topic readiness diagnosis (CLI
  `research chat-doctor` + in-app "Diagnose" button via the `chat_doctor`
  Tauri command).
- **Custom RSS feeds** — user-managed feeds swept on every collect;
  `rss_listings` (G2/SourceForge/Show HN) + `rss_user` sources.
- **Collect fix** — rerun no longer silently skips Reddit (stale one-shot
  localStorage state isolated; explicit "Run without Reddit?" confirm).
