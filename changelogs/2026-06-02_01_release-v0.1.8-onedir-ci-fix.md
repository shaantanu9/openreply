# Release v0.1.8 ‚Äî onedir CI fix + version bump

**Date:** 2026-06-02
**Type:** Infrastructure (Release)

## Summary

Cut the v0.1.8 release and fixed the CI release pipeline, which had been left
inconsistent by the onefile‚Üíonedir sidecar migration (commit `cbef9fe`, still
unpushed). All previous tags (v0.1.3‚Äìv0.1.7) built with the onefile spec, so
CI's `cp dist/openreply-cli <file>` worked; once v0.1.8 carries the onedir spec,
`dist/openreply-cli` is a *directory* and that copy breaks. This change makes the
PyInstaller spec platform-aware and teaches the per-platform release workflows
to stage each form correctly, so a tagged build produces a working, signed,
notarized macOS app (onedir) plus onefile Windows/Linux artifacts.

## Changes

- **`openreply-cli.spec`** ‚Äî platform split: macOS builds **ONEDIR** (the perf fix
  ‚Äî avoids the ~36 s/spawn re-extract under Gatekeeper that caused chat
  timeouts), Windows + Linux build **ONEFILE** (the proven v0.1.x form; no
  Gatekeeper penalty, trivial CI staging). macOS path is byte-identical to
  before; only non-darwin switches to onefile.
- **`scripts/onedir-launcher.sh`** (new) ‚Äî committed launcher that Tauri's
  externalBin copies to `Contents/MacOS/openreply-cli`; it `exec`s the onedir
  engine under `../Resources/`. Committed (rather than generated via an in-YAML
  heredoc) so the shebang can't be mangled by YAML indentation stripping.
- **`.github/workflows/release-mac.yml`** ‚Äî "Stage sidecar" now stages the
  onedir engine to `binaries/openreply-cli-onedir/` and copies
  `scripts/onedir-launcher.sh` to `binaries/openreply-cli-<triple>` (the
  externalBin), mirroring `scripts/publish-mac.sh`. Replaces the broken onefile
  `cp dist/openreply-cli <file>`.
- **`.github/workflows/release-windows.yml` / `release-linux.yml`** ‚Äî the strip
  step now also removes the `openreply-cli-onedir` resource from `tauri.conf.json`
  (it doesn't exist under onefile, so the resource glob would otherwise make
  tauri-bundler error on a missing path).
- **Version bump 0.1.7 ‚Üí 0.1.8** in `tauri.conf.json`, `package.json`,
  `Cargo.toml`, and `Cargo.lock` (skill traps #1 + #17 ‚Äî all version files kept
  in sync).

## Release flow (per tauri-github-release-flow skill)

- Tag `v0.1.8` ‚Üí `release-mac.yml` + `release-windows.yml` + `release-linux.yml`
  build (signed/notarized on mac via repo Apple secrets) and draft on
  `shaantanu9/openreply`.
- `release-promote.yml` flips the public `myind-ai/openreply` release to
  `--draft=false --latest` **only after** the required platform builds succeed ‚Äî
  so a broken build fails safe (stays an unpublished draft).
