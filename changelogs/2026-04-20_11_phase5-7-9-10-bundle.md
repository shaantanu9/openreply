# Phase 5 + 7 + 9 + 10 bundle — cross-topic, export, competitor matrix, research links

**Date:** 2026-04-20
**Type:** Feature

## Summary

Ships the four remaining ROADMAP phases in one coherent bundle:

- **Phase 5 — Cross-Topic Views:** Dashboard leaderboard of highest-Ulwick-scored
  findings across every topic; global findings search; related-topic suggestions
  via Jaccard similarity on source overlap.
- **Phase 7 — Export Brief:** One-click copy to clipboard of the full Minto-pyramid
  synthesis as markdown, standalone hypothesis cards, or a 5-line Slack summary —
  drops directly into Notion / Linear / Docs / DMs.
- **Phase 9 — Competitor Matrix:** Feature × competitor table extracted from the
  report's competitor list, with status cells (has / missing / weakness / unknown)
  and a gap-features callout for greenfield rows no one covers.
- **Phase 10 — Research↔Finding Linker:** Palace-backed semantic matcher that ties
  each finding to the top-K academically similar papers in the corpus. Degrades
  gracefully when palace is unavailable. Renders as a clickable "📚 N research" chip
  on finding cards that opens a modal with paper titles, similarity scores, and
  source-type badges.

All four features are provider-agnostic, lazy-loaded, and silent when empty — they
don't clutter the dashboard on fresh installs but light up the moment data exists.

### Python backend
- `src/reddit_research/research/cross_topic.py` — new module
  - `top_opportunities_across_topics(limit, min_score)` — leaderboard query
  - `search_findings(query, topic_filter, limit)` — global relevance-ranked search
  - `related_topics(topic, limit)` — Jaccard similarity over source breakdown
- `src/reddit_research/research/export_brief.py` — new module
  - `export_markdown(topic, include_hypotheses)` — Minto-structured brief
  - `export_hypothesis_cards(topic)` — bets as standalone markdown
  - `export_slack_summary(topic)` — 5-line DM version
- `src/reddit_research/research/competitors.py` — new module
  - `build_matrix(topic)` — feature × competitor table + gap detection
  - Normalizes feature labels for dedup
- `src/reddit_research/research/research_linker.py` — new module
  - Lazy `finding_research_links` table creation
  - `link_findings_for_topic(topic, k)` — palace-backed matcher, skips embed/ocr
  - `get_links_for_finding(topic, title)` — joined query for modal
  - `get_links_summary(topic)` — per-finding count map for chip rendering
